Introducción

Blitter es una app que permite al usuario escanear un ticket usando la cámara y compartir los gastos con amigos y familiares.

Usando Blitter, ya no tendrás que hacer cálculos complejos para saber cuanto dinero tiene que pagar cada persona cuando vais de compras o a un restaurante juntos.

Simplemente tienes que escanear el ticket usando tu cámara. Blitter reconocerá los productos y sus precios. Después, podrás asignar cada producto a una o más personas. Finalmente, Blitter calculará por ti cuanto tiene que pagar cada persona.

Además, puedes compartir el desglose con tus amigos usando cualquier aplicación.

Desarrollo

El desarrollo del app se puede dividir en cuatro fases.

Fase 1: Especificación de requisitos

Una lista de los requisitos funcionales que se pretendían implementar fue elaborada. Se puede consultar la lista de requisitos aquí:

  • https://github.com/samuelss1996/blitter/blob/master/pre-dev/analysis/requirements.md

Fase 2: Prototipado de la UI

Un prototipo de la interfaz de usuario fue elaborado tomando en consideración los requisitos especificados en la fase previa. El prototipo fue creado uasando Adobe XD

Aquí se muestran algunas capturas de pantalla de este prototipo.

El fichero completo del prototipo se puede descargar aquí:

  • https://github.com/samuelss1996/blitter/blob/master/pre-dev/prototype/blitter.xd

Fase 3: Diseño de software

Una vez definidos los requisitos de la app y como el usuario va a interactuar con la misma en base al prototipo de la UI, era el momento de diseñar la arquitectura general del proyecto. Para elaborarla se usó StarUML2.

Aquí se muestran algunas capturas de los diagramas.

El archivo completo de StarUML2 puede ser descargado aquí:

  • https://github.com/samuelss1996/blitter/blob/master/pre-dev/design/blitter.mdj

Fase 4: Implementación

Basándose en los recursos creados en las fases, se comenzó con la fase de implementación.

La app fue programada para Android 4.1+ usando Android Studio como IDE y Kotlin como lenguaje de programación. El código fuente se puede encontrar en el repositorio de GitHub:

  • https://github.com/samuelss1996/blitter

Fase 5: Publicación

Finalmente, cuando la app estaba totalmente desarrollada, era momento de publicarla en la Google Play Store. Se puede consultar aquí:

  • https://play.google.com/store/apps/details?id=es.soutullo.blitter

La app tuvo un éxito relativo, consiguiendo más de 10.000 descargas y una puntuación promedio de los usuarios de 4,5 de 5.
