Introducción

Este artículo describe el proceso de desarrollo de el portafolio que estás viendo ahora mismo.

Desarrollo

El desarrollo se puede dividir en 6 fases.

Fase 1: Prototipado rápido en papel

Antes de nada, me pareció buena idea hacer un prototipo rápido en papel del portafolio. Con ello, pretendía definir de forma sencilla, rápida y visual cuales iban a ser los contenidos de la web y como organizarlos.

Aquí se muestran algunas imágenes de este prototipo en papel.

La plantilla sobre la que dibujé los prototipos se puede encontrar aquí:

  • https://www.sketchize.com/

Fase 2: Prototipo detallado en Adobe XD

Una vez que tuve una idea general de que información incluir en el portafolio y como organizarla, era momento de averiguar como hacer que fuese bonito y usable (dentro de mis limitados conocimientos, pues no soy diseñador gráfico).

Aquí se muestran algunas capturas del prototipo en Adobe XD.

El archivo de Adobe XD se puede descargar aquí:

  • https://github.com/samuelss1996/portfolio/blob/master/pre-dev/prototype/PortfolioPrototype.xd

Fase 3: Implementación de la vista

Una vez que tuve el prototipo detallado de la vista usando Adobe XD, era el momento de conseguir el mismo resultado pero usando HTML y CSS.

Para ello, se usó Bootstrap para conseguir el layout deseado y Angular para implementar la lógica estrictamente necesaria en este punto del desarrollo (principalmente se definieron los componentes necesarios y se implementó manejo de rutas).

Sin embargo, no se implementó ningún tipo de lógica de back-end. Los datos sobre mí, mis proyectos y actividades y, en general, todo el contenido de la página estaba hardcodeado.

Con ello, prentendía validar de forma empírica si el diseño de la vista que había hecho funcionaba, era usable y satisfacía mis necesidades antes de hacer nada más.

De haber dejado esto para el final, corría el riesgo de darme cuenta demasiado tarde de que el resultado no me satisfacía nada, en cuyo caso sería probable que tuviese que hacer los cambios en muchos lugares.

Fase 4: Implementación de la API

Teniendo ya toda la vista perfectamente definida, implementada y testada, ya sabía exactamente que datos necesitaba obtener desde el back-end.

Para desacoplar de formal total front-end y back-end, decidí crear una API REST. Esta API proporciona en JSON todos los datos necesarios para el funcionamiento de este portafolio.

Por ejemplo, puedes ver la consulta a la API que se ha hecho para mostrarte este artículo en la siguiente URL:

  • Ver consulta a la API

La API está soportada por una base de datos en MongoDB y un servidor HTTP implementado con Node.JS (Express). Se usó la librería mongoose para facilitar la elaboración de consultas a la base de datos.

Fase 5: Integración

Teniendo ya la vista y la API implementadas, era momento de conectar ambas.

En esta fase simplemente se crearon los servicios de Angular necesarios para realizar las consultas al back-end, y se reemplazaron los valores anteriormente hardcodeados por los valores obtenidos mediante estas consultas.

Fase 6: Despliegue

Para desplegar la web se usó heroku, que se integra perfectamente con el sistema de control de versiones (Git) de forma que resulta también sencillo publicar actualizaciones.
