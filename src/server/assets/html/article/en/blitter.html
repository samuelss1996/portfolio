Introduction

Blitter is an app that allows the user to scan a receipt using the camera and share the expenses among friends or family.

Using Blitter you will not have to do complex maths to know how much money each person has to pay when you go shopping or to a restaurant with your friends and family.

You just need to scan the receipt using your camera. Blitter will recognise the products and their prices. Then, you will be able to assign each product to one or more people. Finally, Blitter will calculate for you how much each person has to pay.

Furthermore, you can share the breakdown with your friends using any app.

Development

The development of the app can be divided in four phases.

Phase 1: Requirements specification

A list of the functional requirements intended to be implemented on the app was tailored. The list of requirements can be found here:

  • https://github.com/samuelss1996/blitter/blob/master/pre-dev/analysis/requirements.md

Phase 2: UI Prototyping

A prototype of the user interface was done taking into account the requirements specified in the previous phase. The prototype was created using Adobe XD

Here are some screenshots of this prototype.

The full prototype file can be also downloaded here:

  • https://github.com/samuelss1996/blitter/blob/master/pre-dev/prototype/blitter.xd

Phase 3: Software design

Having already defined the requirements of the app as well as how the user is going to interact with the product based on the UI prototype, it was time to design the overall architecture of the project. To accomplish this, StarUML2 was used.

Here are some screenshots of the diagrams.

The full StarUML2 file can be downloaded here:

  • https://github.com/samuelss1996/blitter/blob/master/pre-dev/design/blitter.mdj

Phase 4: Implementation

Based on all the assets generated on the previous phases, the implementation phase was started.

The app was programmed for Android 4.1+ using Android Studio as the IDE and Kotlin as the programming language. The sources can be found in the GitHub repository:

  • https://github.com/samuelss1996/blitter

Phase 5: Publishing

Finally, when the app was fully developed it was time to publish it in the Google Play Store. It can be found here:

  • https://play.google.com/store/apps/details?id=es.soutullo.blitter

The app was a relative success achieving more than 10,000 downloads and an average user rating of 4.5 out of 5.
