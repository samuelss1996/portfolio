Introduction

This article describes the development process of the portfolio you are browsing at the moment.

Development

The development can be divided in 6 phases.

Phase 1: Paper sketching

Before anything else, I thought it would be a good idea to make a paper sketch of the portfolio. The idea behind this was to define in a simple, quick, visual way what the contents of the web were going to be and how to organize them.

Here are some pictures of this paper sketch.

The template I used to draw on top my sketches can be found here:

  • https://www.sketchize.com/

Phase 2: High-fidelity prototype in Adobe XD

Once I had a general idea on what information to include in the portfolio and how to organize it, it was time to find out how to make it look nice and usable (as best as I could, because I am not a professional graphic designer).

Here are some screenshots of the prototype in Adobe XD.

The Adobe XD file can be downloaded here:

  • https://github.com/samuelss1996/portfolio/blob/master/pre-dev/prototype/PortfolioPrototype.xd

Phase 3: View implementation

Once I had the high-fidelity prototype, it was time to port it to actual code using HTML and CSS.

To achieve that, I used Bootstrap to define the layout and Angular to implement just the very essential logic at this stage of the development (namely I defined the necessary components and I implemented the route handling).

Nonetheless, no back-end logic was implemented. All the data about me, my projects and activities and, in general, all the content of the site was hard-coded.

With this approach, my intention was to validate in an empirical way if the design of the view was correct, usable and fulfilled all my needs before doing anything else.

Had I left these tasks for the end, I would risk to realize too late that the result was not satisfying, in which case it would be more likely that I had to make changes in many places.

Phase 4: API implementation

Having already defined, implemented and tested the view, I knew exactly what data I needed to get from the back-end.

In order to decouple the front-end from the back-end, I decided to create a REST API. This API provides in JSON format all the necessary data for the portfolio to function correctly.

For example, you can check the API request that has been performed to show you this article in the following URL:

  • View the API request

The API is backed by a MongoDB database as well as a HTTP server implemented with Node.JS (Express). The library mongoose was used to facilitate querying the database.

Phase 5: Integration

Having already the view and the API implemented, it was time to connect both.

In this phase I simply created the necessary Angular services to make requests to the back-end. The previously hard-coded values were now replaced with the values returned by these queries.

Phase 6: Deployment

In order to deploy the web heroku was used. It perfectly integrates with the version control system (Git), thus easing the process of publishing updates.
